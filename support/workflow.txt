Plan de cours d'initiation à Git pour débutants en programmation (1 jou

Durée totale : 6-7 heures (y compris des pauses régulières)

I. Introduction à Git (1 heure)
   A. Présentation de Git
   B. Avantages de l'utilisation de Git
   C. Compréhension du concept de système de contrôle de version
   
II. Installation de Git (30 minutes)
   A. Téléchargement et installation de Git
   B. Configuration initiale de Git (nom d'utilisateur, adresse e-mail)
   
III. Concepts fondamentaux de Git (1 heure)
   A. Création d'un nouveau référentiel (repository)
   B. Initialisation d'un référentiel Git
   C. Compréhension des commits
   D. Suivi des modifications (git add, git commit)
   E. Visualisation de l'historique des commits (git log)
   
IV. Branches et fusion (1 heure)
   A. Création de branches (git branch)
   B. Changement de branche (git checkout)
   C. Fusion de branches (git merge)
   D. Gestion des conflits lors de la fusion
   
V. Collaboration avec Git (1,5 heures)
   A. Clonage d'un référentiel distant (git clone)
   B. Synchronisation avec un référentiel distant (git pull, git push)
   C. Travailler avec des branches distantes
   D. Comprendre les pull requests
   
VI. Utilisation avancée de Git (1 heure)
   A. Annulation de modifications (git revert, git reset)
   B. Recherche de modifications (git diff)
   C. Ignorer les fichiers avec .gitignore
   D. Utilisation des alias Git pour des commandes courantes
   
VII. Bonnes pratiques et astuces (30 minutes)
   A. Organisation des référentiels et des branches
   B. Écriture de messages de commit clairs et significatifs
   C. Utilisation de bonnes pratiques de gestion des branches
   D. Utilisation de .gitignore efficacement
   
VIII. Récapitulatif et questions-réponses (30 minutes)
   A. Récapitulation des concepts clés
   B. Répondre aux questions des participants
   C. Partage de ressources supplémentaires (tutoriels, documentations, etc.)

Ce plan de cours vise à fournir aux participants une introduction pratique et complète à Git, 
en mettant l'accent sur les concepts fondamentaux, la collaboration et les bonnes pratiques.

N'oubliez pas d'inclure des exercices pratiques, des démonstrations et des exemples concrets 
pour favoriser l'apprentissage actif et l'engagement des participants.

git config --global user.name "Votre nom"
git config --global user.email "votre@email.com"
git config --global core.editor "code --wait"
git config color.ui true
